# Category page template

Use this skeleton when creating a new `categories/<category>.md`. The pipeline regenerates `landscape.md` and `viz/data.json`; this file is the human-curated layer.

```markdown
---
category: <category-slug>
tools:
  - { name: "Tool One", repo: "org/tool-one" }
  - { name: "Tool Two", repo: "org/tool-two" }
last_reviewed: YYYY-MM-DD
---

# Category Title

## Overview
1–3 short paragraphs framing the category. What problem space, what sub-shapes exist, why someone reaches for this category at all.

## Decision heuristics
- One-liner heuristics for picking *between* tools in this category.
- "If X then prefer Y."
- Cross-category interactions worth flagging here rather than in a tool block.

## Tools

<!-- BEGIN TOOL: org/tool-one -->
### Tool One
<!-- repo: org/tool-one -->

#### What it is
One or two sentences. What problem, in what shape (library / framework / server / CLI / SaaS-with-OSS-core).

#### When to reach for it
- Bullets — when this is the right pick.

#### When not to
- Bullets — failure modes, scaling limits, lock-in, mismatches. Mandatory.

#### How it fits with other tools
Composition vs. competition. e.g. "Pairs with LiteLLM for routing; overlaps with CrewAI on multi-agent coordination."

#### Notable recent developments
- YYYY-MM-DD — short note, link to release/post.

#### Sources
- Docs:
- Repo: https://github.com/org/tool-one
- Notable posts/talks:
<!-- END TOOL: org/tool-one -->

<!-- BEGIN TOOL: org/tool-two -->
### Tool Two
…
<!-- END TOOL: org/tool-two -->
```

## Rules

- Every tool block MUST be wrapped in `<!-- BEGIN TOOL: <repo> -->` / `<!-- END TOOL: <repo> -->` sentinels (with `<repo>` matching the GitHub `org/name`). The pipeline keys edits off these markers.
- The `<!-- repo: <repo> -->` line directly under the heading is a redundant anchor — keep it; it makes the block grep-friendly.
- "When not to" is mandatory.
- Do **not** put star counts, forks, or any numeric metric in the prose. Numbers live in `landscape.md` (generated) and `state/metrics-current.json`. Hand-editing them here breaks the source-of-truth contract.
- Frontmatter `tools:` list is regenerated by the pipeline to mirror the actual headings.
