# viz/

Static d3 treemap visualization of the radar.

## Files

- `treemap.html` — the page. Pure d3 v7 + Tachyons via CDN, no build step.
- `data.json` — current snapshot. Hierarchy is `root → category → tool`. Generated by the `viz:treemap` stage of the sweep pipeline (`RUNBOOK.md` §7.17) from `state/metrics-current.json` + `state/landscape-meta.json`.
- `data-<YYYY-MM-DD>.json` — per-sweep archives.

## Viewing

A plain `file://` open will be blocked by browser CORS when fetching `data.json`. Run any static server from the repo root:

```
python3 -m http.server -d viz 8080
# then open http://localhost:8080/treemap.html
```

## Controls

- **size by** — pick the metric driving leaf area. Default `stars`. Other options: forks, watchers, contributors, open_issues, stars_30d, stars_90d.
- **color by**
  - `deploy status` (default) — our adoption stance, per RUNBOOK §3: 🟢 prod / 🟡 beta / 🔴 deprecated / — untriaged. Color-blind-safe palette; legend pairs hue with the emoji glyph.
  - `velocity` — sequential scale on `stars_30d`.
  - `language` — categorical (`d3.schemeTableau10`); top 8 languages shown in the legend.
  - `upstream status` — the project's lifecycle (`status` field): active / maintenance / dormant / archived.
- **filter** — substring match across name, repo, license, language; non-matching cells dim.
- Click a cell (or focus + Enter) → opens the GitHub repo in a new tab.

## Keyboard

- `/` — focus the filter input
- `Esc` (while in the filter) — clear the filter
- `Tab` — walks through cells; `Enter` / `Space` opens the repo

## Tooltip

Hover (or focus) shows: title, `owner/repo`, two badges (deploy status + upstream status — they're independent axes per RUNBOOK §3), the full metric block, language/license/last-push, curated notes if present, and a hint that clicking opens the repo.

## Data shape (one leaf)

```json
{
  "name": "LangGraph",
  "repo": "langchain-ai/langgraph",
  "deploy_status": "prod",
  "status": "active",
  "stars": 9000, "forks": 1300, "watchers": 120,
  "open_issues": 200, "contributors": 200,
  "stars_30d": 412, "stars_90d": 1180,
  "pushed_at": "2026-04-20",
  "license": "MIT", "language": "Python",
  "notes": "default for stateful agent flows"
}
```

`deploy_status` is one of `prod` / `beta` / `deprecated` / null (null → "untriaged" in the UI). `status` is one of `active` / `maintenance` / `dormant` / `archived`.
